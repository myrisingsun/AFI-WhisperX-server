# Changelog

## v3.0 — Async Queue + Web UI + Diarization (2026-03-18)

### Breaking changes

`POST /transcribe` больше не блокирует соединение до завершения транскрипции.
Теперь возвращает HTTP 202 с `task_id` мгновенно. Результат забирается поллингом через `GET /tasks/<id>`.

Старый синхронный ответ:
```json
{"success": true, "text": "...", "segments_count": 5, ...}
```
Новый ответ на POST:
```json
{"task_id": "uuid", "status": "pending", "poll_url": "/tasks/uuid"}
```

---

### Новые возможности

#### Асинхронная очередь задач

- `POST /transcribe` сохраняет файл, создаёт задачу и возвращает `task_id` (202) — соединение не висит
- Фоновый worker-поток обрабатывает задачи по одной (модель не потокобезопасна, GPU память ограничена)
- `GET /tasks/<id>` — статус задачи + результат по завершении
- `GET /tasks` — список всех задач без поля `result`
- Статусы: `pending → processing → done | error`
- Прогресс внутри задачи: `queued → loading audio → transcribing → aligning → diarizing → saving → done`
- Хранилище задач в памяти, автоматически обрезается до 200 записей
- Temp-файлы именуются `temp_audio_{uuid}_{safe_name}` — без коллизий на Windows

#### Веб-интерфейс (`templates/index.html`)

- Доступен по адресу `http://localhost:5001`
- Загрузка файла + выбор языка прямо в браузере
- Прогресс-бар с названиями этапов обновляется каждые 2 секунды
- Результат: полный текст (с кнопкой "Копировать") и таблица сегментов Start/End/Speaker/Text
- Список прошлых транскрипций с кнопкой скачивания
- Без внешних зависимостей (чистый JS + CSS)

#### Диаризация спикеров (опционально)

- Флаг `--diarize` или переменная окружения `HUGGINGFACE_TOKEN`
- Использует `whisperx.DiarizationPipeline` (pyannote.audio)
- Колонка Speaker в таблице сегментов появляется автоматически
- В `.txt` файле каждый сегмент помечается спикером: `[SPEAKER_00]`
- Поле `"diarization": true/false` в JSON-ответе
- После однократного скачивания моделей работает полностью офлайн

#### Новые endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/tasks/<id>` | Статус и результат задачи |
| `GET` | `/tasks` | Список задач (без поля result) |
| `GET` | `/files/<filename>` | Скачать `.txt` файл |

#### Изменённые endpoints

- `GET /health` — добавлены поля `diarization_enabled`, `queue_depth`
- `GET /` — теперь возвращает HTML веб-интерфейс вместо JSON

### Оптимизации производительности

| Параметр | Было | Стало | Эффект |
|----------|------|-------|--------|
| `compute_type` (GPU) | `float32` | `int8` | Быстрее, вдвое меньше VRAM |
| Кэш alignment-модели | загружалась каждый раз | кэш по языку | -5–15 сек на повторных задачах |

> **Примечание по GPU:** `float16` не поддерживается на Pascal-архитектуре (Quadro P2000 и старше) в ctranslate2. `int8` — оптимальный выбор для этих карт.

---

### Исправления

- Temp-файлы: `re.sub(r'[^\w\-.]', '_', filename)` — защита от спецсимволов Windows в именах файлов
- `PYTHONIOENCODING=utf-8` обязателен при запуске на Windows (cp1251 не поддерживает Unicode-символы в stdout)
- Разрешён конфликт слияния в `README.md`
